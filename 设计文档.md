# 智净大师——任意路径智能文件清理系统

## 需求规格说明书 & 概要设计说明书


## 一、项目概述

### 1.1 项目背景

传统电脑管家类软件（如腾讯电脑管家、微软电脑管家）的文件清理功能几乎都默认锁定在C盘，用户无法自由指定其他磁盘或文件夹进行扫描清理。而现实生活中，开发者的项目依赖（如node_modules、target目录）动辄占用数十GB，设计师的素材库堆积大量旧版文件，普通用户的下载目录可能积累了数年未动的安装包和临时文件。这些散落在D盘、E盘乃至外接硬盘中的文件，才是磁盘空间真正的“沉默杀手”。

与此同时，市面上已有的深度清理工具普遍存在“重扫轻确认”的问题——它们往往替用户决定什么该删、什么该留，缺乏透明、可控的二次确认机制。用户要么不敢用（怕误删重要文件），要么用了之后追悔莫及。

“智净大师”正是为解决这一痛点而生。它支持用户指定任意磁盘路径进行扫描，基于可配置的智能引擎识别出推荐清理的文件与空文件夹，并以清晰的列表呈现。用户可单击打开文件所在文件夹、双击定位到具体文件，通过人工二次勾选确认后再执行清理。整个过程透明、安全、可控，真正做到“你看到的是什么，删掉的就是什么”。

### 1.2 产品愿景

做一个让用户完全放心的“文件瘦身助手”。不替用户做决定，而是把清理的决策权完整交还给用户——系统只负责找出“可能需要清理的文件”，最终删不删、删哪些，全凭用户自己确认。

### 1.3 核心价值

| 价值维度 | 描述 |
| :--- | :--- |
| **任意路径自由扫描** | 突破传统工具仅限C盘的限制，用户可指定任意磁盘、任意文件夹进行扫描，包括外接硬盘和U盘 |
| **透明化的推荐清理列表** | 清晰呈现每个被推荐删除文件的路径、大小、类型和推荐理由，按类别分组展示，一目了然 |
| **便捷的文件定位功能** | 列表中的每个文件支持“打开所在文件夹”和“打开文件位置”，用户可在清理前快速核验文件 |
| **人工二次确认机制** | 所有清理项默认不勾选，由用户逐项或批量勾选确认后执行，杜绝误删风险 |
| **空文件夹专项清理** | 独立识别并呈现目标路径下的所有空文件夹，一键清理无用目录结构，恢复文件系统的整洁 |

## 三、概要设计说明书

### 3.1 总体架构

“智净大师”采用 **模块化前后端分离架构**，前端负责用户交互与结果展示，后端核心引擎负责文件扫描、分类识别与清理执行。整体分为四层：

- **用户界面层（UI Layer）**：桌面GUI（基于Tauri或Qt构建），包含路径管理面板、扫描结果列表、文件预览、清理配置等组件。命令行CLI提供文本界面和脚本化能力。
- **业务逻辑层（Service Layer）**：扫描任务调度器负责管理多线程扫描任务的创建、暂停、继续和取消；结果聚合器将扫描结果按分类归组、去重、排序；清理执行器负责校验清理列表、执行删除操作并生成报告。
- **核心引擎层（Core Engine）**：文件遍历器支持高性能递归目录遍历，可配置深度、过滤条件等。文件分类引擎基于规则库对每个文件进行多维度分类打分，确定推荐等级。空文件夹检测器独立遍历目录树识别空文件夹，提供合并建议。重复文件检测器（可选模块）基于分层哈希策略快速定位内容相同的文件。
- **基础设施层（Infrastructure）**：提供跨平台文件系统访问抽象（统一Windows/Linux/macOS差异）、进程文件占用检测（联动Restart Manager API或lsof）、系统文件管理器调用（`explorer /select,`、`open -R`、`xdg-open`）等能力。

### 3.2 技术选型

| 技术领域 | 选型 | 理由 |
| :--- | :--- | :--- |
| **跨平台桌面框架** | Tauri v2 (Rust) | 内存安全且性能卓越，生成的安装包体积小（< 5MB），自带系统托盘和自动更新支持，前端可灵活选用Web技术栈 |
| **前端UI** | React + TypeScript + Vite | 组件化开发效率高，支持虚拟列表（处理10万级数据），生态丰富 |
| **核心引擎语言** | Rust | 零成本抽象带来的C/C++级别性能，内存安全保证没有空指针和缓冲区溢出风险，适合处理大量文件I/O操作 |
| **文件遍历** | `walkdir` crate (Rust) 或 `jwalk` (并行遍历) | 成熟稳定，支持递归遍历、深度控制、符号链接处理，跨平台兼容 |
| **文件哈希** | `blake3` crate | 比SHA256快近10倍，适合大文件的高速哈希计算 |
| **数据库（本地）** | SQLite (rusqlite) | 存储扫描历史、路径收藏、自定义规则、清理记录等，无需服务器 |
| **进程检测** | Windows: Restart Manager API; macOS: `lsof`; Linux: `/proc/*/fd` | 检测文件被占用情况，防止删除正在使用的文件 |
| **命令行解析** | `clap` crate (Rust) | 声明式参数解析，自动生成帮助信息，支持子命令 |

### 3.3 核心模块设计

#### 3.3.1 文件分类引擎

这是系统“智能推荐”的核心。引擎基于规则库（默认+用户自定义），对每个文件进行多维度打分。

**规则库结构**：每条规则包含“匹配条件”（文件名模式/扩展名/文件大小范围/文件时间阈值/路径模式）、“分类标签”（临时文件/缓存文件/日志文件/构建产物/旧备份/大文件/下载残留之一）、“推荐权重”（0-100，表示该文件属于该分类的确信程度）、“是否保护”（标记为True则该文件不可被清理）。

**匹配流程**：文件遍历器产出文件流后，引擎流水线依次经过各规则组进行匹配。同一文件可能命中多条规则，因此结果表以文件路径为键聚合所有命中分类及最高权重。最终，推荐列表按“总权重降序+文件大小降序”排列展示。

**内置规则示例**：
- 规则`TMP_001`：匹配`*.tmp`、`*._mp`、`~*.*`，归入“临时文件”，权重90。时间阈值：最后修改时间>7天则权重+5。不保护。
- 规则`LOG_001`：匹配`*.log`、`*.log.*`，归入“日志文件”，权重60。若文件>100MB则权重+30。不保护。
- 规则`BUILD_001`：匹配路径包含`node_modules`、`target`、`.venv`、`build`等关键词的目录，归入“构建产物”，权重95。保护条件：目录是否在版本控制忽略列表中——若是则不保护；若项目最近7天有活跃commit则降权。
- 规则`TEMP_PATH`：匹配位于`%TEMP%`、`%TMP%`、`/tmp`、`/var/tmp`等系统临时目录下的所有文件，归入“系统临时文件”，权重70。时间>3天则加至85。不保护。
- 规则`SYS_PROTECT`：匹配位于`C:\Windows\`、`/System/`、`/boot/`等系统关键路径下的文件，归入“系统文件”，标记为保护。

#### 3.3.2 文件遍历与扫描调度

采用生产者-消费者模型。生产者线程（文件遍历器）将发现的文件路径通过有界通道（`crossbeam channel`）发送给消费者线程池。每个消费者线程负责执行规则匹配，将结果推送至结果收集器。

**可配置参数**：递归深度（默认无限制）、是否追踪符号链接（默认否）、是否包含隐藏文件（默认是）、文件大小过滤（默认>1KB）、排除路径模式（支持glob）。

暂停功能通过向遍历器发送停止信号实现，继续时从中断点重启遍历。

#### 3.3.3 空文件夹检测器

独立于文件扫描引擎，采用“后序遍历”策略——先检查子文件夹，再检查父文件夹。仅当子文件夹被确认空（或被清理后变为空）时，才将父文件夹标记为候选。这避免了重复清理，也使得结果更加准确。

**合并建议逻辑**：若路径`A/B/C`为空，且`A/B`仅包含`C`（空文件夹），则系统建议以`A/B`为单位进行清理（即一并清理B和C）。

#### 3.3.4 文件定位与系统交互

利用各平台的原生能力实现文件定位功能。

- **Windows**：调用`explorer /select, "<文件完整路径>"`，资源管理器打开并高亮文件。对于目录，调用`explorer "<目录路径>"`。
- **macOS**：调用`open -R "<文件路径>"`在Finder中定位文件；`open "<目录路径>"`打开目录。
- **Linux**：调用`xdg-open "<目录路径>"`打开目录；文件定位则根据可用性调用`nautilus --select`、`dolphin --select`或`thunar`等。

#### 3.3.5 清理执行与安全防护

**默认不勾选+二次确认**：扫描结果展示后，所有复选框初始状态为未勾选。用户点击“清理选中”按钮时，弹出确认对话框，明确显示即将删除的文件数和总大小。用户需手动点击“确认删除”按钮，不可使用回车键直接触发。

**清理模式**：移至回收站（默认模式，利用各平台回收站API）将文件移至系统回收站，用户可从回收站还原。永久删除使用标准文件删除操作，文件系统层面仍可恢复。安全擦除则在删除前用随机数据覆写文件1-3次（可配置），再执行删除，确保无法恢复。

**防误删保护**：
- 系统关键路径（`C:\Windows\`、`/System/`等）硬编码，不可修改，其下文件强制保护。
- 调用操作系统API检测文件是否被其他进程占用：Windows使用Restart Manager API，Linux检查`/proc/*/fd`，macOS使用`lsof`。占用文件显示“文件正在使用”状态，不推荐清理。
- 用户可在列表中右键将任意文件标记为“保留”，标记后该文件变为不可勾选状态。

### 3.5 数据模型设计

**扫描历史表（scan_history）**：记录每次扫描的元数据。字段包括扫描ID（UUID）、扫描路径、扫描开始与结束时间、发现候选文件数、发现候选总大小（字节）、空文件夹数、用户最终清理文件数、清理释放大小、扫描配置JSON（含递归深度、文件大小阈值等）。

**路径收藏表（favorite_paths）**：字段包括收藏ID、路径、备注名称、添加时间。

**自定义规则表（custom_rules）**：字段包括规则ID、规则名称、匹配模式类型（glob或正则）、匹配值、分类标签、推荐权重、时间阈值天数、是否保护、是否启用、创建时间。

**清理记录表（cleanup_log）**：字段包括记录ID、关联扫描ID、清理时间、清理文件路径、文件大小、文件原分类、删除方式（回收站或永久或安全擦除）、操作结果（成功或失败）、失败原因。

### 3.6 界面设计

整体采用左右分栏布局，左侧为导航与控制面板，右侧为主内容区。

- **导航栏（左侧）**：依次为“开始扫描”、“扫描结果”、“空文件夹”（带数字角标）、“清理历史”、“设置”等入口。下方显示路径收藏列表，点击可快速切换。
- **主内容区（右侧）**：
  - **路径选择页**：顶部大输入框（输入或粘贴路径），右侧“浏览”按钮和“开始扫描”主按钮。输入框下方为最近扫描路径列表。中间区域为扫描范围配置（递归深度滑块、排除路径、文件大小过滤等折叠面板）。底部为磁盘空间概览卡片。
  - **扫描结果页（核心）**：顶部为结果概览条（“已发现 X 个推荐文件，共 Y GB，空文件夹 Z 个”）。主体为分组表格视图。左侧边栏用于按分类筛选（全部分类、临时文件、缓存、日志、构建产物等），每项显示文件数和大小。右侧表格每行包含：复选框、文件/目录图标、文件名、原始路径（灰色小字）、大小、修改时间、分类标签（彩色徽章）、操作按钮（“打开文件夹”和“打开文件位置”图标按钮）。表格支持多列排序和Ctrl/Shift批量选择。
  - **空文件夹页**：以树状结构展示空文件夹层次，每个节点显示复选框、文件夹图标、文件夹名称、完整路径和所属层级。顶部提供“全选”、“展开全部”、“折叠全部”快捷操作。
- **确认清理对话框**：模态弹窗，展示即将删除的文件数、总大小、各类别汇总和删除模式选择（回收站/永久/安全擦除，默认回收站）。底部显著区分“确认删除”（红色按钮）和“取消”（灰色按钮）。支持勾选“不再询问本次”（本次会话内）。
- **设置页**：包含清理规则管理（展示默认规则和自定义规则，支持增删改启用）、时间阈值全局默认值、安全路径保护列表和主题切换。

### 3.7 安全与风险控制

| 安全措施 | 具体实现 |
| :--- | :--- |
| **默认不勾选** | 所有扫描结果初始复选框为空，需用户手动勾选 |
| **二次确认** | 清理前弹出汇总确认对话框，不可被回车键跳过 |
| **系统路径保护** | 内置不可删除路径列表硬编码于引擎层，不受用户配置影响 |
| **进程占用检测** | 调用Restart Manager API (Windows) 或 lsof (macOS/Linux)，占用文件标记为“使用中”不可清理 |
| **回收站模式（默认）** | 删除文件默认移至回收站，用户可从系统回收站恢复误删文件 |
| **操作日志** | 每次清理操作均记录于SQLite中，支持追溯 |
| **测试模式** | 提供“仅预览”模式，不实际删除，便于初次使用者熟悉流程 |

## 四、总结

“智净大师”重新定义了文件清理工具的设计理念——它不是替用户做决定的“黑盒管家”，而是一个透明、可控、安全的“文件瘦身助手”。通过支持任意路径扫描、智能分类推荐、便捷的文件定位和严格的人工二次确认机制，让用户在完全掌控的前提下高效释放磁盘空间。模块化的架构设计和基于Rust的高性能引擎，确保系统在处理万级文件时依然流畅响应。无论是普通用户清理下载目录，还是开发者清理构建产物，都能在这个工具中找到最高效、最安心的使用体验。
